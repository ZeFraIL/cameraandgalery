# Класс: FileInfo

## 1. Общая информация
*   **Имя класса:** `FileInfo`
*   **Тип:** Обычный класс (Модель данных / POJO)
*   **Назначение:** Этот класс служит «контейнером» или «моделью данных». Его единственная цель — хранить информацию о конкретном медиафайле (фото или видео) в структурированном виде.
*   **Взаимодействие с другими компонентами:**
    *   Используется `DatabaseHelper` для представления строки из базы данных.
    *   Используется `ViewInfoAboutMyFiles` и его адаптером для отображения данных в списке.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `name` | `String` | Хранит имя файла. | Конструктор и `getName()`. |
| `date` | `String` | Хранит дату создания файла. | Конструктор и `getDate()`. |
| `time` | `String` | Хранит время создания файла. | Конструктор и `getTime()`. |
| `uri` | `String` | Хранит путь (адрес) файла. | Конструктор и `getUri()`. |
| `type` | `String` | Хранит тип: "image" (изображение) или "video" (видео). | Конструктор и `getType()`. |

## 3. Методы класса
### Имя метода: `FileInfo` (Конструктор)
*   **Тип:** `public`
*   **Возвращаемое значение:** Нет (Конструктор)
*   **Параметры:** `name`, `date`, `time`, `uri`, `type` (все `String`).
*   **Что делает метод:** Инициализирует новый экземпляр класса, присваивая переданные значения полям класса.
*   **Когда вызывается:** Когда нужно создать новый объект, представляющий файл, обычно после чтения из базы данных.

### Имена методов: `getName`, `getDate`, `getTime`, `getUri`, `getType` (Геттеры)
*   **Тип:** `public`
*   **Возвращаемое значение:** `String` (значение соответствующего поля)
*   **Что делает метод:** Просто возвращает значение приватного поля вызывающей стороне.
*   **Зачем нужно:** Для соблюдения инкапсуляции (скрытия данных).

## 4. Жизненный цикл
*   Не применимо (это обычный Java-класс, а не Activity).

## 5. Взаимодействие с интерфейсом (UI)
*   Напрямую с UI не взаимодействует. Он только предоставляет данные, которые интерфейс в итоге отобразит.

## 6. Взаимодействие с другими компонентами
*   **База данных:** Используется для преобразования строк БД в Java-объекты.
*   **UI компоненты:** Используется `RecyclerView`, чтобы знать, какой текст или изображение показать.

## 7. Общая логика класса
Этот класс — простой «Объект передачи данных» (DTO). Он не выполняет сложных вычислений. Он просто хранит данные, чтобы мы могли передавать один объект `FileInfo` вместо 5 отдельных строк.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте, что `FileInfo` — это **библиотечная карточка**. На этой карточке записаны название книги, дата поступления, место на полке и тип обложки. Сама карточка ничего не «делает», но с ней гораздо удобнее работать, чем пытаться запомнить все детали о книге по отдельности.
