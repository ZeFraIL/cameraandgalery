# Класс: DatabaseHelper

## 1. Общая информация
*   **Имя класса:** `DatabaseHelper`
*   **Тип:** Обычный класс (наследует `SQLiteOpenHelper`)
*   **Назначение:** Управляет локальной базой данных (SQLite). Отвечает за создание файла БД, создание таблиц, обновление структуры и вставку данных.
*   **Взаимодействие с другими компонентами:**
    *   `MainActivity` использует его для сохранения данных о новых файлах.
    *   `ViewInfoAboutMyFiles` использует его для чтения списка файлов.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение |
| :--- | :--- | :--- |
| `DATABASE_NAME` | `String` | Имя файла базы данных (`all_info.db`). |
| `DATABASE_VERSION` | `int` | Номер версии (нужен для обновлений). |
| `TABLE_NAME` | `String` | Название таблицы. |
| `COLUMN_...` | `String` | Названия колонок (имя, дата, время, путь, тип). |
| `SQL_CREATE_ENTRIES` | `String` | Команда SQL для создания таблицы. |

## 3. Методы класса
### Имя метода: `onCreate`
*   **Что делает метод:** Выполняет SQL-команду для создания таблицы.
*   **Когда вызывается:** Автоматически системой при первом обращении к базе данных.

### Имя метода: `insertFileInfo`
*   **Что делает метод:**
    1. Открывает базу для записи.
    2. Упаковывает данные в объект `ContentValues`.
    3. Вставляет запись в таблицу.
    4. Закрывает соединение.
*   **Когда вызывается:** После того, как пользователь сделал фото или записал видео.

## 4. Жизненный цикл
*   Управляется системой Android (жизненный цикл базы данных).

## 5. Взаимодействие с интерфейсом (UI)
*   Прямого взаимодействия нет. Работает «под капотом».

## 6. Взаимодействие с другими компонентами
*   **SQLite Database:** Является мостом между кодом Java и реальным файлом в памяти телефона.

## 7. Общая логика класса
Этот класс централизует всё «делопроизводство». Вместо того чтобы писать сложные SQL-запросы по всему приложению, мы просто вызываем понятные методы, например `insertFileInfo`.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте `DatabaseHelper` как **Библиотекаря**. Когда вы хотите зарегистрировать новую книгу (файл), вы не идете сами писать в большую амбарную книгу. Вы даете данные Библиотекарю, а он точно знает, в какую книгу (базу), на какую страницу (таблицу) и в какой столбец их записать.
