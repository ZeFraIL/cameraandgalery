# نشاط: Start

## 1. معلومات عامة
*   **اسم الفئة:** `Start`
*   **النوع:** Activity (شاشة)
*   **الدور في التطبيق:** هذه هي "Splash Screen" (شاشة البداية) أو نقطة الدخول للتطبيق. تعرض شاشة مؤقتة (عادةً مع شعار) قبل نقل المستخدم إلى الجزء الرئيسي من التطبيق.
*   **التفاعل مع المكونات الأخرى:** تبدأ `MainActivity` بعد تأخير زمني.

## 2. المتغيرات (حقول الفئة)
لا تحتوي هذه الفئة على متغيرات عامة. كل المنطق موجود داخل الطريقة `onCreate`.

## 3. طرق (Methods) الفئة
### اسم الطريقة: `onCreate`
*   **ماذا تفعل الطريقة:**
    1. تقوم بإعداد تخطيط الشاشة (Layout).
    2. تفعل وضع "Edge-to-Edge" (المحتوى خلف شريط الحالة).
    3. تبدأ مؤقتاً تنازلياً (`CountDownTimer`) لمدة 2000 مللي ثانية (ثانيتين).
*   **متى تُستدعى:** عند فتح التطبيق لأول مرة.

### اسم الطريقة: `onFinish` (داخل المؤقت)
*   **ماذا تفعل الطريقة:**
    1. تنشئ `Intent` (طلب انتقال) لـ `MainActivity`.
    2. تستدعي `startActivity(go)` لتغيير الشاشات.
*   **متى تُستدعى:** تلقائياً بعد وصول المؤقت إلى الصفر.

## 4. دورة الحياة (Lifecycle)
*   **`onCreate()`**: تُستدعى عند إنشاء النشاط. هنا يتم إعداد واجهة المستخدم وبدء المؤقت.

## 5. التفاعل مع واجهة المستخدم (UI)
*   **التخطيط:** يستخدم `R.layout.activity_start`.
*   **التكيف:** يضبط الهوامش (Padding) لضمان عدم تغطية عناصر النظام (مثل الساعة) لمحتوى التطبيق.

## 6. التفاعل مع المكونات الأخرى
*   **MainActivity:** ينتقل إلى هذه الشاشة بعد ثانيتين.

## 7. المنطق العام للفئة
يبدأ التطبيق من هنا. يرى المستخدم شاشة لمدة ثانيتين. خلف الكواليس، يقوم مؤقت بالعد التنازلي. بمجرد انتهاء الوقت، ينتقل التطبيق تلقائياً إلى الشاشة الرئيسية.

## 8. شرح مبسط
**الشرح بكلمات بسيطة:**
فكر في فئة `Start` كأنها **غرفة انتظار** أو **مقدمة فيلم**. عندما تدخل السينما، لا يبدأ الفيلم فوراً؛ أولاً ترى مقدمة قصيرة أو شعاراً. بعد بضع ثوانٍ، يبدأ العرض الرئيسي. هذه الفئة هي تلك المقدمة التي تستغرق ثانيتين وتقول "مرحباً" قبل أن يبدأ العمل الحقيقي.
