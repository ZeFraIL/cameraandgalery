# فئة: DatabaseHelper

## 1. معلومات عامة
*   **اسم الفئة:** `DatabaseHelper`
*   **النوع:** فئة عادية (ترث من `SQLiteOpenHelper`)
*   **الدور في التطبيق:** تدير هذه الفئة قاعدة البيانات المحلية (SQLite). هي مسؤولة عن إنشاء ملف قاعدة البيانات، وإنشاء الجداول، وتحديث الهيكل، وإدخال البيانات.
*   **التفاعل مع المكونات الأخرى:**
    *   تستخدمها `MainActivity` لحفظ معلومات الملفات الجديدة.
    *   تستخدمها `ViewInfoAboutMyFiles` لقراءة قائمة الملفات.

## 2. المتغيرات (حقول الفئة)
| الاسم | النوع | الغرض |
| :--- | :--- | :--- |
| `DATABASE_NAME` | `String` | اسم ملف قاعدة البيانات (`all_info.db`). |
| `DATABASE_VERSION` | `int` | رقم الإصدار (يساعد في التحديثات). |
| `TABLE_NAME` | `String` | اسم الجدول الذي يخزن المعلومات. |
| `COLUMN_...` | `String` | أسماء الأعمدة (الاسم، التاريخ، الوقت، المسار، النوع). |
| `SQL_CREATE_ENTRIES` | `String` | أمر SQL لإنشاء الجدول. |

## 3. طرق (Methods) الفئة
### اسم الطريقة: `onCreate`
*   **ماذا تفعل الطريقة:** تنفذ أمر SQL لإنشاء الجدول.
*   **متى تُستدعى:** تلقائياً بواسطة أندروايت في أول مرة يحاول فيها التطبيق الوصول إلى قاعدة البيانات.

### اسم الطريقة: `insertFileInfo`
*   **ماذا تفعل الطريقة:**
    1. تفتح قاعدة البيانات للكتابة.
    2. تغلف القيم في كائن `ContentValues`.
    3. تدرج البيانات في الجدول.
    4. تغلق الاتصال.
*   **متى تُستدعى:** يدوياً من `MainActivity` بعد التقاط صورة أو فيديو.

## 4. دورة الحياة (Lifecycle)
*   تدار بواسطة النظام (دورة حياة قاعدة البيانات).

## 5. التفاعل مع واجهة المستخدم (UI)
*   لا يوجد تفاعل مباشر مع واجهة المستخدم. تعمل "خلف الكواليس".

## 6. التفاعل مع المكونات الأخرى
*   **قاعدة بيانات SQLite:** هي الجسر بين كود الجافا والملف الحقيقي في ذاكرة الهاتف.

## 7. المنطق العام للفئة
تجمع هذه الفئة كل عمليات "التدوين". بدلاً من كتابة كود SQL معقد في كل مكان في التطبيق، نقوم ببساطة باستدعاء طرق سهلة مثل `insertFileInfo`.

## 8. شرح مبسط
**الشرح بكلمات بسيطة:**
فكر في `DatabaseHelper` كأنه **أمين مكتبة**. عندما تريد تسجيل كتاب (ملف) جديد في المكتبة، فأنت لا تذهب وتكتب في سجل التسجيل الكبير بنفسك. أنت تعطي المعلومات لأمين المكتبة، وهو يعرف بالضبط في أي كتاب (قاعدة بيانات)، وفي أي صفحة (جدول)، وفي أي عمود يجب تسجيلها.
