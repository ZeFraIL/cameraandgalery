## 🎯 Цель проекта
•	**Какую задачу решает приложение:** Позволяет пользователю делать фотографии и записывать видео, автоматически сохраняя метаданные файлов (имя, дата, время, путь) в локальную базу данных для последующего просмотра.
•	**Почему эта задача важна:** Упрощает организацию медиафайлов и предоставляет быстрый доступ к информации о созданных файлах в одном месте.
•	**Целевая аудитория:** Пользователи, которым необходимо вести учет созданного медиаконтента.
________________________________________
## 📌 Требования к приложению
### Функциональные требования
•	Создание фото через системную камеру.
•	Запись видео через системную камеру.
•	Сохранение информации о файле (имя, дата, время, URI) в SQLite.
•	Просмотр списка всех сохраненных записей.
•	Просмотр деталей файла (превью фото или воспроизведение видео) во всплывающем диалоге.
•	Открытие системной галереи.

### Нефункциональные требования
•	**Производительность:** Быстрая загрузка списка файлов из БД с использованием RecyclerView.
•	**Удобство использования:** Простой интерфейс с четырьмя основными кнопками на главном экране.
•	**Надёжность:** Обработка разрешений (Permissions) и проверка наличия камеры.

## 🧠 Общая архитектура
•	**Выбранный подход:**
–	MVC (Model-View-Controller)

•	**Почему выбран именно он:**
Для небольшого проекта такой подход обеспечивает четкое разделение между данными (Model), интерфейсом (View) и логикой управления (Controller/Activity), не перегружая код лишними абстракциями.

•	**Основные компоненты системы:**
- **Model:** DatabaseHelper, FileInfo.
- **View:** XML-разметки (activity_main, item_file_info).
- **Controller:** MainActivity, ViewInfoAboutMyFiles.

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
**Роль:** Главный экран приложения.
**Ответственность:** Управление жизненным циклом, обработка нажатий кнопок, запуск Intents для камеры/видео, запрос разрешений.
**Основные методы:**
- `onCreate()` — инициализация UI и лаунчеров.
- `dispatchTakePictureIntent()` — запуск камеры для фото.
- `dispatchTakeVideoIntent()` — запуск камеры для видео.
- `saveFileInfoToDatabase()` — запись данных в БД.

### 📌 Класс: DatabaseHelper
**Роль:** Слой данных.
**Почему используется:** Для работы с локальной базой данных SQLite, хранения структуры таблиц и выполнения запросов.

### 📌 Класс: FileInfo
**Роль:** Модель данных.
**Описание:** Содержит поля `name`, `date`, `time`, `uri`, `type` для представления информации о медиафайле.

### 📌 Класс: ViewInfoAboutMyFiles
**Роль:** Экран просмотра списка.
**Ответственность:** Загрузка данных из БД, отображение их в RecyclerView через адаптер, показ диалога с деталями.
________________________________________
## 🔄 Диаграмма работы приложения
1. Пользователь нажимает "Take Photo".
2. Открывается системная камера.
3. После съемки данные (путь к файлу и время) сохраняются в SQLite через `DatabaseHelper`.
4. Пользователь переходит в "View Info", где видит список всех записей.
5. При нажатии на элемент списка открывается диалог с предпросмотром.

## 🎨 UI/UX анализ
•	**Почему интерфейс сделан именно так:** Использована плиточная структура кнопок для быстрого доступа к основным функциям.
•	**Какие принципы использованы:**
–	**Простота:** Минимальное количество действий для достижения цели.
–	**Логичность:** Группировка действий по смыслу.
•	**Что можно улучшить:** Добавить поддержку темной темы и возможность удаления записей.
________________________________________
## ⚙️ Работа с потоками
•	**Используются ли:**
- В текущей версии операции с БД и UI выполняются преимущественно в основном потоке (Main Thread), что допустимо для малых объемов данных.
•	**Как предотвращаются зависания (ANR):** Использование эффективных методов Glide для загрузки изображений, которые работают в фоновых потоках.
________________________________________
## 💾 Работа с данными
•	**Где хранятся данные:** Локальная БД SQLite (all_info.db) и внутренняя память приложения (для медиафайлов).
•	**Почему выбран этот способ:** Обеспечивает автономность и быструю выборку данных без интернета.
________________________________________
## 🌐 Работа с сетью
•	В данной версии работа с сетью не предусмотрена (приложение полностью офлайн).
